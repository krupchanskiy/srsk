## Пакет A — Ежедневный ритуал сверки ✅
Коммит: «Пакет A», миграция 204, кэш v13.
- **Главная:** быстрые действия (+Приход/Расход/Перевод), карточка «Давность сверки» по
  реальным счетам (точка-цвет: сегодня зелёный / N дн. жёлтый-красный, несвежее сверху,
  клик → сверка), минусы кликабельны → лента счёта.
- **Счета:** клик по строке → лента ДДС, кнопка «Сверить» → сверка, цвет давности
  чекпоинта, «Ни разу не сверено».
- **Сверка:** deep-link `?account=` + автофокус на первом номинале, клавиатурный поток
  (Enter → следующее поле, значение выделяется), success-состояние + серия чекпоинтов
  без расхождений (готовит критерий запуска из runbook §4).
- Проверено: deep-link открыл счёт с автофокусом, серия показана; давность и клики верны.

## Пакет B — Журнал, которому доверяют (ДДС) ✅
Коммит: «Пакет B», миграция 205, кэш v14.
- Поиск по сумме (±, режим счёта/статьи), пресеты периода (Сегодня/Неделя/Месяц/Всё),
  фильтр по статье (новый режим «проводки»: счёт ИЛИ статья), итог по выборке
  (приход/расход по валютам), экспорт CSV всей выборки, автор операции в развороте,
  «Повторить» из разворота (расход/приход/пожертвование — форма по образцу), deep-link
  `?op=` (разворот+подсветка+автоскролл).
- Проверено: итог «Приход ₹380 700 · Расход ₹170 000», режим статьи (колонка «Счёт»,
  свой итог), поиск «170000» → 1 строка.

## Пакет C — День заезда (Участники) ✅

  кнопка сводки на месте.

## Пакет D — Cutover (Справочники) ✅
Коммит: «Пакет D», миграция 207, кэш v16.

  Бейдж «действует» не показан только потому, что в тест-данных курсов нет.

## Что осталось тебе (осознанно)
- **3.4 групповой платёж** (участник в строке платежа, сц.3 ТЗ) — трогает логику формы,
  просил не делать без надзора.
- **8.2 массовый импорт статей** — то же.
- **7.2 клик по статье в аналитике → ДДС с фильтром** — оказался НЕ низкорисковым:
  отчётные RPC (`fin_get_retreat_report`, `fin_get_summary_report`) возвращают статьи
  только по имени, без `category_id`. Нужна серверная правка — добавить `id` в отчёт,
  тогда deep-link `dds.html?category=<id>` заработает (фильтр по статье в ДДС уже готов,
  пакет B). Полчаса работы, но с изменением RPC — не ночью.
