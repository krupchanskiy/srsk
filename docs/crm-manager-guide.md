# Инструкция менеджера по продажам ретритов

## Как попадают заявки

Гость заполняет форму на сайте rupaseva.com → в CRM автоматически создаётся сделка со статусом **Новая заявка**. Система назначает менеджера по очереди (round-robin).

Ты получаешь нового гостя с его контактными данными: имя, телефон, email, telegram.

---

## Воронка продаж

Каждая сделка проходит через статусы. Статус — это **степень готовности гостя**, а не чеклист действий.

| Статус | Что это значит | Что делать |
|--------|---------------|------------|
| **Заявка** (lead) | Гость оставил заявку, мы ещё не связывались | Связаться в течение 24 часов |
| **Связались** (contacted) | Поговорили, гость заинтересован | Ответить на вопросы, выслать информацию |
| **Оплачено** (paid) | Получена оплата | Помочь с визой, трансфером, билетами, размещением |
| **Готов** (ready) | Всё подготовлено к заезду | Ждать заезда, финальные уточнения |
| **Завершена** (completed) | Ретрит окончен, гость уехал | Собрать обратную связь |
| **Отменена** (cancelled) | Гость отказался | Узнать причину, предложить следующий ретрит |

**Важно:** переключай статус сразу, когда ситуация изменилась. Это помогает видеть реальную картину по всем сделкам.

## Чеклист по каждому гостю

Помимо статуса, для каждого гостя нужно выяснить и зафиксировать:

### Проживание
- **В ШРСК** — размещаем в наших комнатах (уточнить тип: общая/отдельная)
- **Партнёрская гостиница** — бронируем в одной из гостиниц, с которыми работаем
- **Самостоятельно** — гость решает сам, нам ничего делать не нужно

### Питание
- **В ШРСК** — ест в нашей столовой (по умолчанию для проживающих у нас)
- **Самостоятельно** — питается сам

### Трансфер и рейсы
- **Нужен из аэропорта** — встречаем. ОБЯЗАТЕЛЬНО узнать: номер рейса, дату и время прилёта, аэропорт
- **Нужен в аэропорт** — отвозим. ОБЯЗАТЕЛЬНО узнать: номер рейса, дату и время вылета, аэропорт
- **Не нужен** — добирается сам
- Система считает **время приезда в ШРСК** = прилёт + 4 часа (дорога). Если приезд **до 10:00** → ранний заезд
- Система считает **время выезда из ШРСК** = вылет − 6 часов (дорога + запас). Если выезд **после 13:00** → поздний выезд

### Виза
- **Нужна помощь** — помогаем с приглашением или консультацией
- **Не нужна** — у гостя уже есть виза или он не нуждается (граждане Индии, e-Visa и т.д.)

### Билеты
- **Уже куплены** — зафиксировать даты прилёта/вылета
- **Ещё не куплены** — напомнить ближе к дедлайну
- **Не нужны** — гость в Индии

### Дополнительно (выясняется в процессе)
- **Духовный учитель** — записать имя
- **С кем хочет жить рядом** — имя друга/знакомого (передаётся менеджеру по расселению)
- **Особые потребности** — диета, здоровье, мобильность

## Финальная передача на расселение

Перед передачей гостя менеджеру по расселению, в карточке должно быть заполнено:

| Что | Где в карточке | Обязательно |
|-----|---------------|-------------|
| Оплата получена (полная или предоплата) | Вкладка «Оплаты» | Да |
| Проживание (где живёт) | Чеклист | Да |
| Питание (с нами или нет) | Чеклист | Да |
| Рейс прилёта (номер, дата, время) | Блок «Рейсы» | Да, если нужен трансфер |
| Рейс вылета (номер, дата, время) | Блок «Рейсы» | Да, если нужен трансфер |
| Ранний заезд / поздний выезд | Блок «Рейсы» | Автоматически |
| С кем хочет жить | Допполя | Желательно |

Когда всё заполнено — нажми **«Передать на расселение»**. Система создаст задачу для менеджера по расселению со всеми данными. После этого ты увидишь в карточке, какую комнату гостю назначили.

## Ежедневная работа

### Утро (10 минут)
1. Открой **Канбан** → посмотри новые заявки (колонка «Новая заявка»)
2. Каждую новую заявку — связаться сегодня
3. Посмотри **Мои задачи** — что нужно сделать сегодня

### В течение дня
1. После каждого контакта → **Записать контакт** в боковой панели сделки (выбери тип: звонок/WhatsApp/Telegram/email)
2. Если статус изменился → переключи его
3. Получил оплату → добавь во вкладку «Оплаты»

### Важные правила
- **Не оставляй заявку без ответа больше 24 часов**
- **Записывай каждый контакт** — даже короткий. Если ты заболел, коллега должен понять где дела
- **Проверяй историю гостя** — если человек уже был на наших ретритах, это видно в карточке сделки. Используй это: «Рады видеть вас снова!»

## Частые ситуации

### Гость купил билет, но не внёс предоплату
Статус: **Контакт установлен** или **Счёт отправлен** (зависит от того, отправляли ли счёт). Билеты — это отдельный чеклист, не статус воронки. Зафиксируй даты билета в заметках и напомни про оплату.

### Гость живёт в Индии
Трансфер и билеты могут быть не нужны. Отметь «не нужен» в чеклисте. Виза тоже не нужна. Сконцентрируйся на проживании и оплате.

### Гость хочет приехать с семьёй
Создай отдельные сделки на каждого взрослого. Детей запиши в заметки к сделке основного контакта.

### Гость передумал
Переведи в статус **Отменена**, укажи причину. Через месяц можно написать и предложить следующий ретрит.

### Повторный гость
Система покажет историю — на каких ретритах был, какой был статус. Поприветствуй лично, предложи те же условия или улучшенные.

## Где что находится

| Задача | Где в системе |
|--------|--------------|
| Посмотреть все сделки | CRM → Сделки (таблица с фильтрами) |
| Посмотреть воронку визуально | CRM → Канбан |
| Открыть конкретную сделку | Клик на имя гостя |
| Записать звонок/сообщение | Сделка → боковая панель → «Записать контакт» |
| Добавить оплату | Сделка → вкладка «Оплаты» → «+ Добавить» |
| Посмотреть историю гостя | Сделка → вкладка «Инфо» → блок «История гостя» |
| Создать задачу себе | Сделка → вкладка «Задачи» → «+ Добавить» |
| Создать сделку вручную | CRM → Сделки → «+ Новая сделка» |
