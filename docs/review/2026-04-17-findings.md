### 1.1. RLS и запросы к Supabase

#### Обход авторизации через прямой URL (🔴 Critical)

45 страниц имеют запись в `pagePermissions` (layout.js), но не проверяют право на самой странице. Меню фильтруется, но прямой URL даёт доступ к HTML+JS со всеми данными (RLS на БД защищает содержимое, но UI загружается целиком, а часть запросов может вернуть данные через роли).

| ID | Severity | Location | Description | Fix idea |

#### `.single()` на таблицах с RLS (🟠 High)

`.single()` возвращает ошибку 406 если запрос вернул 0 строк — в том числе когда RLS скрыла строку. `.maybeSingle()` возвращает `null`, что безопаснее и даёт информацию «скрыто vs не существует» через `error.code`.

| RLS-02 | 🟠 High | `js/pages/person.js:70` | `vaishnavas` | `.single()` на SELECT по id. RLS скрытие → 406 «not found» вместо пустого UI. | `.maybeSingle()` + «не найдено / нет прав» UI. |

| DATE-05 | 🟠 High | `js/pages/stock-requests.js:901` | `created_at` | Отображение времени создания — смещение UI относительно ожидаемого. | `DateUtils.formatDateTime()`. |
